# Кросс-валидация слоя 8.5 (субсидии): две независимые реализации

Слой субсидий посчитан **дважды, независимо**, разными людьми и на разных
стеках, и результаты сведены получатель-в-получателя. Это сильнейшая из
доступных проверок методики: совпадение двух независимых расчётов подтверждает
реализацию надёжнее любого числа модульных тестов одной кодовой базы.

| | Реализация A (эта, FastAPI) | Реализация B (Django MVP, ветка `django-mvp`) |
|---|---|---|
| Источник значений индикаторов | лист «Риск_получатели» книги (готовый расчёт аналитиков) | пересчёт с нуля из листа «Данные» |
| Стек | FastAPI + SQLAlchemy + PostGIS | Django + GeoDjango + pandas |
| Территориальный охват | все 3413 получателей книги | только 1944 текущей Алматинской области |

Оба стека поднимались на одном файле-источнике
`Риск_субсидии_Алматинская.xlsx` (SHA-256
`6494b5f4…95cb0e9`, сверен с манифестом).

## Результат

Сравнивались **1944 получателя**, общих для обеих реализаций (те, что
привязаны к текущей Алматинской области). БИН сошлись полностью: 0 лишних с
любой стороны.

| Что сверялось | Результат |
|---|---|
| **Уровень риска** (low/medium/high/critical) | **1944 из 1944 совпали, 0 расхождений** |
| Индикатор s2 «повторность» | бит-в-бит (макс Δ 0.001 — округление) |
| Индикатор s3 «аффилированность» | бит-в-бит (Δ 0.000) |
| Индикатор s4 «процессные аномалии» | бит-в-бит (макс Δ 0.001) |
| Индикатор s5 «выбросы сумм» | бит-в-бит (макс Δ 0.001) |
| Индикатор s1 «концентрация» | совпал у 1929, **разошёлся у 15** (макс Δ 0.048) |
| Итоговый балл R | медиана \|Δ\| = 0.015 на шкале 0–100 |

Полное совпадение уровней риска — главное: именно уровень красит карту,
наполняет распределение на панели и попадает в рейтинг районов. Ни один объект
не поменял цвет между реализациями.

## Единственное объяснимое расхождение: s1 на дореформенной границе

Формула концентрации у обеих реализаций одна и та же:

```
s1 = max( (доля_в_районе − 0.05) / 0.45 ; доля_в_области / 0.10 ),  clamp[0;1]
```

Расходится только «доля в области», и только у 15 получателей, у которых
доминирует именно областная нога `max(...)`:

- **Реализация A** берёт `доля_в_области` из книги, где знаменатель — вся
  **дореформенная** Алматинская область: 3413 получателей, 67,5 млрд ₸.
- **Реализация B** считает по **текущей** области: 1944 получателя, 56,73 млрд ₸.

Меньший знаменатель у B делает долю больше, а с ней и s1. Отсюда систематический
сдвиг: балл B у этих 15 чуть выше. Это не ошибка ни одной из реализаций — это
тот же водораздел «до / после реформы 08.06.2022», что проходит через весь
территориальный слой (см. `territory-reconciliation.md`, § 4). Книга 8.5
измеряет концентрацию внутри исторической области; реализация B — внутри
текущей.

На уровни риска сдвиг не повлиял: ни один из 15 не пересёк порог.

## Вывод

Методика расчёта риска субсидий **подтверждена независимым воспроизведением**.
Расхождение локализовано, объяснено и сводится к одному сознательному
методологическому выбору (какая граница области берётся за знаменатель
концентрации), а не к дефекту кода. На отображаемые пользователю уровни риска
расхождение не влияет.

Как воспроизвести: поднять оба стека на одном файле-источнике, выгрузить из
каждого `(БИН, risk_score, risk_level)` по привязанным к области получателям и
сджойнить по БИН.
