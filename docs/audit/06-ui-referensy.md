# Аудит UI-референсов

Источник: восемь скриншотов в `C:\Users\erbot\Downloads\ДЭР\photo_2026-07-19_*.jpg`.
Разобраны визуально 2026-07-20.

> **Важно.** Все числа на референсах — демонстрационные (45,2 млрд ₸, 708 объектов,
> 12 456 субъектов, «данные актуальны на 04.07.2026»). Они **не** являются
> контрольными значениями и не должны попасть в код как фикстуры. Реальные
> значения берутся только из книг ДЭР. Из референсов заимствуется **структура и
> поведение**, не содержимое.

## 1. Две визуальные темы

Референсы распадаются на два набора, и это осознанное разделение, а не разнобой:

| Набор | Экраны | Фон | Сайдбар |
|---|---|---|---|
| Тёмный | Карта (2 скриншота) | почти чёрный, тёмная подложка карты | тёмно-синий `#0f2744`, активный пункт — синяя заливка |
| Светлый | Дашборд, Импорт, Отчёты, Граф, Администрирование | `#f1f5f9` | тот же тёмно-синий |

Сайдбар одинаков в обеих темах — это якорь оболочки. На карте в топбаре есть
переключатель темы (иконка луны), то есть тёмный режим — режим приложения, а не
отдельный дизайн. **Вывод для реализации:** одна система токенов, две темы;
карта по умолчанию открывается в тёмной, остальные экраны — в светлой,
переключатель глобальный.

## 2. Оболочка

**Левый сайдбар** (одинаков везде): логотип «Карта рисков» (на карте — с
подписью `Preview`), заголовок группы `НАВИГАЦИЯ`, пункты в неизменном порядке:

1. Дашборд
2. Карта
3. Данные (импорт)
4. Отчёты
5. Граф связей
6. Администрирование

Внизу — `© 2026 Акимат`.

**Топбар, светлые экраны:** поиск «Поиск по реестрам системы…» с подсказкой
горячей клавиши `⌘K`, колокольчик с бейджем непрочитанных, карточка пользователя
(аватар с инициалами, ФИО, роль под ним).

**Топбар, карта:** поиск «Поиск по БИН, ФИО, объекту…», кнопки `Экспорт`,
`Фильтры`, `Источники` (подсвечена янтарным — активное состояние), переключатель
темы, роль пользователя.

Расхождение плейсхолдеров поиска между наборами — скорее всего, черновая
несогласованность референса. Приводим к единому поиску с областью действия:
плейсхолдер по ТЗ должен покрывать БИН/ИИН/ФИО/организацию/объект/договор/территорию.

## 3. Экран «Карта»

**Подложка:** тёмная. Границы соседних регионов подписаны латиницей
(`JETISU REGION`, `JAMBYL REGION`, `ALMATY PROVINCE`) — то есть базовая карта
англоязычная, а слой районов подписывается по-русски. Для закрытого контура
англоязычная подложка от внешнего CDN недопустима; нужен self-hosted стиль.

**Полигоны районов** обведены цветом уровня риска, заливка полупрозрачная.
Цвета читаются как: зелёный (низкий), янтарный (средний), красный (высокий),
тёмно-красный (критический), серый (нет данных).

**Popup территории** — карточка с полями строго в таком порядке:

```
Уйгурский район                (заголовок)
Үйғыр ауданы                   (название на казахском, вторичным цветом)
Адм. центр      с. Чунджа
Население       74 000 чел.
Площадь         13 900 км²
Бюджет          ₸13 млрд        (акцентный синий)
Уровень риска   Средний         (цвет уровня)
                Нажмите для приближения   (подсказка)
```

Отсюда следуют требования к модели: у территории обязаны быть казахское
название, адм. центр, население, площадь. Это ровно то, что должно прийти из
`Административно_территориальный_слой.docx` и книги численности населения.

**Правая аналитическая панель**, вкладки `Дашборд` / `Риски` / `ТОП`:

- `ОСВОЕНИЕ БЮДЖЕТА 2024` — список сфер с процентом и полосой прогресса
  (АПК, Строительство, Цифровизация, Образование, Здравоохранение, ЖКХ, Дороги,
  Соц. сфера). Полоса окрашена по уровню, не по значению процента.
- `РИСКИ ПО СФЕРАМ` — чипы вида `budget ₸752 млн`, окрашенные по уровню риска.
  Заметно, что часть подписей осталась техническими кодами (`agro`, `oehs`,
  `procurement`) вперемешку с русскими («Строительство») — в реализации все
  подписи должны быть человекочитаемыми и локализованными.
- `СТАТУСЫ МАТЕРИАЛОВ` — три числовых плитки: Превенция / В ЕРДР / Завершено.
  Это криминогенная часть ТЗ; названия статусов берём из ТЗ, не из референса.

**Левая панель фильтров** (выдвижная, с крестиком закрытия):

| Блок | Контрол |
|---|---|
| `ПЕРИОД` | пилюли 2022 / 2023 / 2024 + две даты «с»–«по» + кнопка `Сравнить периоды` |
| `ТЕРРИТОРИЯ` | выпадающий список, дефолт «Все районы» |
| `ОТРАСЛЬ` | выпадающий список, дефолт «Все отрасли» |
| `УРОВЕНЬ РИСКА` | чекбоксы с цветными точками: Низкий, Средний, Высокий, Критический, Нет данных |
| `СУММА, МЛРД ₸` | два числовых поля диапазона |
| `СТАТУС ОБЪЕКТА` | выпадающий список |

Внизу закреплены `Сбросить` и `Применить` (синяя основная).

Важно: «Нет данных» — **полноправный фильтруемый уровень** наравне с
остальными, а не служебное состояние. Это подтверждает требование ТЗ никогда не
прятать неполноту.

**Список тематических слоёв** — отдельная панель с заголовком
`ТЕМАТИЧЕСКИЕ СЛОИ (5 / 10)`, то есть счётчик «включено / всего». Видимые
пункты: Административно-территориальный, Социально-экономический, Бюджетный,
Государственные закупки, Хозяйствующие субъекты. Первые три включены. Всего
слоёв десять — остальные ниже по прокрутке (инфраструктура, субсидии, риски и
т. д.).

## 4. Экран «Дашборд»

Хлебные крошки `Главная › Аналитическая панель`, заголовок «Аналитическая
панель», подзаголовок с областью и отметкой актуальности данных
(«данные актуальны на …»). Справа — выбор периода (`Q2 2024`), `Экспорт`,
`Обновить`.

**Восемь KPI-карточек, сетка 4×2.** Каждая: иконка в левом верхнем углу,
дельта к прошлому периоду справа (зелёная со стрелкой вверх / красная вниз),
крупное значение, название, мелкая уточняющая подпись:

| # | Значение | Название | Подпись |
|---|---|---|---|
| 1 | 45,2 млрд ₸ | Объём бюджета региона | план на 2024 г. |
| 2 | 12,85 млрд ₸ | Государственные закупки | 42 заказчика |
| 3 | 3,42 млрд ₸ | Субсидии и иная поддержка | 187 получателей |
| 4 | 47 | Инфраструктурные проекты | в реализации |
| 5 | 12 456 | Хозяйствующие субъекты | зарегистрировано |
| 6 | 127 | Высокий / критический риск | объектов |
| 7 | 8,75 млрд ₸ | Сумма финансовых рисков | оценочно |
| 8 | 23 | Аналит. материалы / меры | актуальных |

Набор KPI точно ложится на пять слоёв данных + сводные риски. Подпись
«оценочно» у суммы финансовых рисков — прямой намёк, что величина расчётная и
требует раскрытия методики в тултипе.

**Ниже два виджета:**
- «Динамика финансовых показателей», линейный график, «млрд тенге, поквартально»,
  переключатель годов 2022/2023/2024, ось X — `КВ1'23 … КВ2'24`;
- «Объекты по уровню риска», кольцевая диаграмма, подзаголовок «N объектов
  всего», легенда-таблица с числом на каждый уровень, включая `Нет данных`.

Ниже видны начала ещё двух виджетов (обрезаны) — по ТЗ это рейтинги территорий
и проблемные объекты.

## 5. Экран «Данные (импорт)»

Заголовок «Загрузка и импорт данных», подзаголовок «Мастер загрузки в 3 шага».
Горизонтальный степпер с нумерованными кружками:

1. Загрузка файла → 2. Сопоставление столбцов → 3. Предпросмотр и подтверждение

**Шаг 1:** блок `ТИП ЗАГРУЖАЕМЫХ ДАННЫХ` — шесть плиток-переключателей
(Государственные закупки, Бюджетные данные, Субсидии и поддержка,
Хозяйствующие субъекты, Инфраструктурные проекты, Соц-экон. показатели),
выбранная обведена синим. Ниже — зона перетаскивания: «Перетащите файл или
нажмите для выбора», подпись «Excel (.xlsx, .xls), CSV, JSON, GeoJSON ·
Макс. 50 МБ». Кнопка `Далее` неактивна, пока файл не выбран.

**Правая колонка `ИСТОРИЯ ЗАГРУЗОК`:** список последних импортов, в каждом —
имя файла, «N строк · дата», статусный бейдж (`ОК` зелёный /
`Предупрежд.` янтарный). Это подтверждает требование журнала импортов с
версионированием.

Шесть типов данных на референсе против пяти книг комплекта: «Соц-экон.
показатели» — отдельный тип, а инфраструктура на референсе одна плитка, хотя в
книге 8.6 две несвязанные популяции. Расхождение фиксируется в
`assumptions-and-gaps`.

## 6. Экран «Отчёты и экспорт»

Подзаголовок «Формирование аналитических справок и выгрузка данных».
Восемь карточек-шаблонов сеткой 4×2, у каждой иконка, название, описание,
кнопка `Сформировать`:

1. **Сводный отчёт по региону** — общая аналитика по области с рейтингами и динамикой
2. **Отчёт по территории** — детальный анализ района: бюджет, закупки, организации, риски
3. **Справка по организации** — полное досье: риск, договоры, связи, история изменений
4. **Отчёт по объекту/проекту** — карточка проекта с финансами, участниками, индикаторами
5. **Анализ по отрасли** — сравнительная аналитика и риски по выбранной отрасли
6. **Отчёт по категории риска** — объекты с критическим и высоким уровнем
7. **Рейтинги территорий и отраслей** — сравнительный рейтинг по ключевым индикаторам
8. **Перечень высокорисковых объектов** — реестр с расшифровкой факторов

Формат выгрузки на референсе не показан — берём из ТЗ (Word/Excel/PDF, карта в
растр/PDF).

## 7. Экран «Граф связей»

**Левая панель:** блок `Типы связей` — чекбоксы с цветными точками:
руководитель, учредитель, поставщик, подрядчик, получатель, со-получатель.
Блок `ТИПЫ УЗЛОВ` — легенда: Организация (синий), Физ. лицо (серый),
Договор (янтарный), Субсидия (зелёный). Внизу `Сбросить` и `Экспорт`.

**Канва:** светлая, с точечной сеткой. Управление зумом в левом верхнем углу
(приблизить, отдалить, вписать). Узлы-организации — прямоугольные карточки с
типовой аббревиатурой внутри (`ТОО`, `ДГЗ`, `СУБ`), рамка окрашена по уровню
риска, в правом верхнем углу — точка-индикатор риска; подпись под узлом
(«РегионСтрой», «ДГЗ-2024-1842», «Субсидия ИП-2024»). Физлица — круглый аватар
с подписью ФИО. Рёбра подписаны типом связи и окрашены в цвет типа; часть рёбер
пунктирные, часть сплошные — предположительно различие подтверждённой и
предполагаемой связи, что согласуется с требованием ТЗ хранить `confidence`.

## 8. Экран «Администрирование»

Заголовок «Администрирование системы», кнопка `+ Добавить пользователя`.
Вкладки: `Пользователи` | `Справочники` | `Критерии риска` | `Журнал действий`.

Таблица пользователей, колонки: Ф.И.О., ЛОГИН (моноширинный), РОЛЬ (круглый
чип с инициалами роли + подпись), ТЕРРИТОРИЯ, ПОСЛЕДНИЙ ВХОД (моноширинная
дата-время), СТАТУС (бейдж `Активен` зелёный / `Неактивен` серый), ДЕЙСТВИЯ
(иконки «редактировать» и «блокировать»).

Роли на референсе: **Аналитик**, **Просмотр**, **Руководитель** — совпадает с
ТЗ (там ещё Администратор). Территория в строке пользователя («Алматинская
обл.», «Карасайский р-н», «Все районы») подтверждает, что RBAC
**территориально ограничен**, а не только ролевой.

## 9. Извлечённая палитра

Значения сняты с референсов приблизительно и подлежат проверке на контраст
WCAG 2.1 AA перед фиксацией в токенах.

| Роль | Ориентир |
|---|---|
| Сайдбар / фон тёмной темы | `#0f2744` … `#0b1a2e` |
| Фон светлых экранов | `#f1f5f9` |
| Панели/карточки | `#ffffff` |
| Акцент (кнопки, ссылки, активный пункт) | `#2563eb` / `#3b82f6` |
| Риск: низкий | зелёный `#22c55e` |
| Риск: средний | янтарный `#eab308` … `#f59e0b` |
| Риск: высокий | красный `#ef4444` |
| Риск: критический | тёмно-красный `#991b1b` |
| Риск: нет данных | серый `#9ca3af` |
| Подсветка «Источники» | янтарная обводка |

**Ограничение доступности.** На референсе уровень риска в ряде мест передан
только цветом (точка в чекбоксе, обводка полигона, точка на узле графа). По
требованию ТЗ (WCAG 2.1 AA) цвет не может быть единственным носителем смысла:
добавляем текстовую подпись уровня, различимые паттерны заливки полигонов и
форму/иконку индикатора на узлах графа. Пара «высокий `#ef4444`» и
«критический `#991b1b`» на тёмном фоне различима плохо — потребуется развести
их по светлоте или добавить штриховку критическому.

## 10. Что референсы задают, а что нет

**Задают:** состав и порядок пунктов навигации; трёхшаговый мастер импорта с
историей загрузок; восемь шаблонов отчётов; набор фильтров и их порядок;
структуру popup территории; вкладки правой панели карты; вкладки админки;
типы узлов и связей графа; территориальный RBAC; счётчик тематических слоёв.

**Не задают (берём из ТЗ):** переключатель `Списком | На карте` — на
референсах его нет вообще, хотя ТЗ требует; карточки сущностей; расшифровку
факторов риска; экраны сравнения периодов; мобильные раскладки; форматы
экспорта; состояния загрузки/ошибки/пустоты.

Отсутствие переключателя «Списком | На карте» на референсах — самое крупное
расхождение между референсами и ТЗ. Приоритет по правилам проекта у ТЗ:
переключатель реализуем, встраивая его в топбар экрана карты рядом с
`Фильтры`.
