## 2.2 Методика Risk Score (дословно, лист «Методика Risk Score»)

```
ШАГ 1. Значение индикатора v — приведение к безразмерному v ∈ [0;1] по градуированным
       порогам. Если поле пусто либо источник не подключён — avail = ЛОЖЬ, индикатор
       НЕ участвует ни в числителе, ни в знаменателе.
ШАГ 2. S_raw   = Σ (w_i × v_i)   по доступным индикаторам i
ШАГ 3. W_avail = Σ (w_i)         по доступным индикаторам i
ШАГ 4. S_norm  = 100 × S_raw / W_avail            → 0..100
ШАГ 5. K — коэффициент значимости:
       Тип A: K = 1,00 + 0,15×I_стоимость + 0,15×I_уровень
              I_стоимость = 1, если стоимость проекта в верхнем квартиле выборки
              I_уровень   = 1, если проект республиканского уровня
       Тип B: K = 1,00 + 0,15×I_опасность + 0,15×I_ответственность
              I_опасность       = 1, если 1–2 класс опасности
              I_ответственность = 1, если 1 уровень ответственности
       Диапазон K: 1,00 … 1,30
ШАГ 6. Risk Score = min(100 ; S_norm × K)
ШАГ 7. completeness = 100 × W_avail / W_total   (W_total = 110 для типа A, 90 для типа B)
ШАГ 8. Уровень риска:
       если completeness < 50 %                              → «серый»
       иначе если ошибка в данных (окончание раньше начала)  → «серый»
       иначе если Risk Score ≥ 75 → КРИТИЧЕСКИЙ (тёмно-красный)
       иначе если Risk Score ≥ 50 → ВЫСОКИЙ (красный)
       иначе если Risk Score ≥ 25 → СРЕДНИЙ (жёлтый)
       иначе                      → НИЗКИЙ (зелёный)
```

**Шкала уровней:** НИЗКИЙ 0–24 · СРЕДНИЙ 25–49 · ВЫСОКИЙ 50–74 · КРИТИЧЕСКИЙ 75–100 · СЕРЫЙ — уровень не присваивается.

> Обратите внимание: пороги 8.6 (**25/50/75**) отличаются от порогов 8.5 (**35/55/75**). Слои не согласованы между собой.

**Статус весов (дословно):** «Веса ЧЕРНОВЫЕ и назначены экспертно. Калибровка возможна только на размеченной выборке подтверждённых нарушений, которой сейчас нет. До калибровки веса — параметр администратора (ТЗ п.14, п.98), а не константа кода.»

## 3.2 Формула (лист «Формула», дословно)

```
ШАГ 1. v ∈ 0..1 по порогам листа «Веса». Если источник не подключён или значение пустое —
       avail = ЛОЖЬ, индикатор НЕ участвует ни в числителе, ни в знаменателе.

ШАГ 3. W_avail = Σ (w_i)         по всем доступным индикаторам i
ШАГ 4. risk_score  = 100 × S_raw / W_avail            → 0..100
ШАГ 5. completeness = 100 × W_avail / W_total,  где W_total = 110
ШАГ 6. Уровень риска:
       если Категория A                → «критический» (тёмно-красный), балл не важен
       иначе если completeness < 50 %  → «серый» (ТЗ 7.3)
       иначе если risk_score ≥ 75      → «критический»
       иначе если risk_score ≥ 50      → «высокий»
       иначе если risk_score ≥ 25      → «средний»
       иначе                           → «низкий»
```
