## Решения, которые нельзя молча отменить

**Отсутствие данных — «не измерено», а не ноль.** Неизмеренный индикатор не
входит ни в числитель, ни в знаменатель; если не измерено ничего, балла нет
вовсе (`None`). Серый уровень значит «не знаем», а не «риска нет», и остаётся
полноправным уровнем в фильтрах, легенде и агрегатах. Закреплено тестами по
всему коду.

**Организация с максимальным баллом 93.3 официально имеет уровень «нет
данных»**, а критическими становятся организации с баллом 42 и ниже по
жёсткому правилу категории A. Выглядит противоречиво, но это требование
методики: балл при полноте 41 % не основание для вывода, а признак
лжепредприятия — основание. Есть тест, который это защищает: соблазн
«починить» сортировкой по баллу велик.

**Границы взяты из OpenStreetMap (ODbL), а не из комплекта.** Приложенный
набор GADM отражает деление до реформы 2022 года, не содержит пяти
действующих единиц и распространяется под лицензией, запрещающей
перераспространение, вопреки заявленному в архиве public domain.

**Слои живут на разных территориальных уровнях.** 8.3 — вся республика по
областям, 8.4/8.5 — районы Алматинской области, 8.7 — без территории вовсе.
Карта иерархическая: на уровне республики бюджетный слой, при зуме — закупки
и субсидии. Каждый слой объявляет уровни, на которых существует; недоступность
объясняется пользователю текстом.

**Две популяции слоя 8.6 разделены навсегда.** Проекты ГЧП и заключения
экспертизы — разные таблицы под общим супертипом. Общего ключа нет: проверены
БИН (0 из 1014 и 0 из 4842), пересечение наименований (0 из 1266 × 4781),
участники. Есть тест: наивная связка по номеру даёт 198 ложных совпадений.

**КАТО отсутствует у всех районов.** Единственный официальный ключ стыковки
недоступен, связывание идёт по названиям через справочник алиасов. Свёртка
снимает тип единицы, регистр и графику, но опечатки («Сарканский» против
«Саркандский») намеренно НЕ лечит: правило, склеивающее похожие строки, рано
или поздно склеит разные территории.

**`useSearchParams` в этом проекте не используется.** Связка с границей
Suspense давала бесконечную заглушку и в разработке, и в продакшн-сборке.
Состояние читается из `window.location` с подпиской на `popstate`.

**Территория красится по худшему из измеренных объектов, а не по доле
проблемных.** Доля разваливается на малых выборках: три получателя субсидий и
один высокий дают «33 %». Максимум не занижает риск никогда — единственная
безопасная сторона ошибки для карты рисков. Рядом с цветом всегда показывается
распределение по уровням, иначе один критический объект из трёхсот неотличим от
трёхсот критических. Разбор — в `docs/assumptions-and-gaps.md`.

## Известные ограничения — это границы источников, не недоделки

- `budget_facts` пуста: 74 831 сырая строка не грузится, агрегация внутри
  книги невоспроизводима;
- `persons`, `organization_person_roles`, `addresses` пусты — данных нет;
- учредителей нет ни в одной книге, тип связи в графе даёт честный ноль;
- сущности «аналитические материалы» не существует, показатель говорит
  «нет данных» с объяснением;
- слой 8.7 обеспечен источниками на 41 %, работают 4 индикатора из 13;
- откат импорта снимает актуальность версии, но не восстанавливает прежние
  значения полей: построчной истории в схеме нет;
- три «серых» договора слоя 8.4 — зафиксированный дефект книги
  (`B1=0` при `W_avail=45` вместо 60), воспроизведён как есть.

Полный разбор — `docs/assumptions-and-gaps.md`.

## Грабли этой среды

1. **PowerShell 5.1 портит UTF-8.** Никогда не править файлы через
   `Set-Content`/`Out-File` — читает как ANSI и записывает двойным
   кодированием. Только инструменты редактирования. Скрипты `.ps1` должны
   быть в UTF-8 **с BOM**, иначе кириллица в них ломается.
2. **Heredoc `<<` в PowerShell не работает** — это синтаксис bash.
   Многострочный текст писать во временный файл.
3. **`next start` молча держит порт.** Старый процесс не умирает от
   `pkill`; убивать по PID через `Get-NetTCPConnection -LocalPort 3001`,
   иначе сервер отдаёт устаревшую сборку и диагностика уходит не туда.
4. **Сервер не перезапускается сам** после правки кода — дважды приводило к
   404 и 500 на исправленном коде. Проверять запросом, а не тем, что «код
   написан».
5. **Имена файлов 8.6 и 8.7 в Unicode NFD** — сравнивать только после
   `unicodedata.normalize("NFC", …)`.
6. **Скриншоты подвисают** на WebGL-канве: проверять через дерево
   доступности и содержимое DOM.

## Как передавать проект

```powershell
git bundle create riskmap.bundle --all              # только код
powershell -File scripts\dump-database.ps1          # полный дамп
powershell -File scripts\dump-database.ps1 -NoPersonalData
powershell -File scripts\restore-database.ps1 -DumpPath <файл>
```

Полный дамп содержит **персональные данные**: БИН и ИИН 3668 организаций и
3413 получателей субсидий. Дампы лежат в `dist/` и исключены из git.

## Стиль работы, о котором договорились

- Проверять запуском, а не заявлять по факту написания кода. Непроверенное
  помечать непроверенным — конфигурация Docker помечена прямо в README.
- Не подгонять код под ожидаемый ответ и не ослаблять тесты: расхождение
  фиксировать явно и объяснять.
- Не выдумывать данные. Отсутствие показывать пользователю словами.
- Сообщения коммитов — по-русски, объясняют «почему», а не перечисляют «что».
- Комментарии в коде — по-русски, объясняют причину решения.
