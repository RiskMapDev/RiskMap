# ИАС «Интерактивная карта рисков»

Информационно-аналитическая система оценки социально-экономических и
криминогенных рисков в регионе. Шесть источников-книг превращаются в
пространственную витрину с объяснимой оценкой риска: карта, единый список
объектов, аналитическая панель, граф связей, мастер импорта, отчёты в трёх
форматах.

Система **работает на реальных данных**: PostgreSQL 18.4 с PostGIS 3.6,
37 таблиц-моделей, шесть моделей риска, 13 841 оценённый объект в пяти витринах
и 16 266 связей графа.

Принцип, на котором держится всё остальное:

> **Отсутствие данных — это «не измерено», а не ноль.** Неизмеренный индикатор
> не входит ни в числитель, ни в знаменатель; если не измерено ничего, балла
> нет вовсе. Серый уровень означает «мы не знаем», а не «риска нет», и остаётся
> полноправным уровнем в фильтрах, легенде и агрегатах.

## Быстрый старт

Два пути. Выберите тот, что подходит вашей машине.

### Путь A. Docker — одна команда

Требуется Docker и Docker Compose.

```bash
cp .env.example .env
# заполните JWT_SECRET:
#   python -c "import secrets; print(secrets.token_urlsafe(48))"

docker compose up -d
docker compose exec backend python -m alembic upgrade head
docker compose exec backend python -m scripts.seed_access
```

Интерфейс — `http://localhost:3000`, API и схема — `http://localhost:8100/docs`.

> **Эта конфигурация написана, но не проверена запуском.** На машине, где
> проект собирался, Docker отсутствует. Всё остальное в этом файле проверено
> выполнением; путь через Docker — нет. Отнеситесь к первому запуску
> соответственно и сверяйтесь с [`docs/deployment.md`](docs/deployment.md).

### Путь B. Без Docker, Windows — проверенный

Не требует прав администратора: PostgreSQL с PostGIS разворачивается из
ZIP-бинарников в пользовательский каталог.

```powershell
# Разовая установка сервера базы — см. docs/deployment.md, § 1
powershell -ExecutionPolicy Bypass -File scripts\setup-windows.ps1
```

Скрипт создаёт кластер и базу, генерирует `.env` со свежим секретом, ставит
зависимости, применяет миграции и заводит учётные записи.

### Данные

База поднимается пустой. Дальше одно из двух.

**Если вам передали дамп:**

```powershell
powershell -File scripts\restore-database.ps1 -DumpPath dist\riskmap-....dump
# затем обязательно смените пароли:
cd backend; python -m scripts.seed_access --reset-passwords
```

**Если у вас есть книги комплекта ДЭР** — укажите путь к ним в `SOURCE_DATA_DIR`
и загрузите:

```powershell
cd backend
python -m scripts.source_manifest build      # SHA-256 источников
python -m scripts.load_territories
python -m scripts.load_layers --layer all
python -m scripts.build_relations
```

Импорт **никогда не пишет** в каталог источников и проверяет их неизменность по
контрольным суммам.

### Запуск

```powershell
cd backend  ; python -m uvicorn app.main:app --port 8100
cd frontend ; npm run build ; npx next start -p 3001
```

Проверка живости: `http://127.0.0.1:8100/ready` — отвечает состоянием базы и
версией PostGIS.

## Передача проекта другим

```powershell
# только код, без данных — безопасно
git bundle create riskmap.bundle --all

# или с данными, если получатель вправе их обрабатывать
powershell -File scripts\dump-database.ps1                    # полный дамп
powershell -File scripts\dump-database.ps1 -NoPersonalData    # без ПДн
```

> **Полный дамп содержит персональные данные:** БИН и ИИН 3668 организаций и
> 3413 получателей субсидий, их наименования и суммы выплат. Это не
> обезличенный набор. Передавайте только тем, кто вправе обрабатывать такие
> сведения, и только по защищённому каналу.
>
> Ключ `-NoPersonalData` даёт дамп со схемой, справочниками и границами, но с
> пустыми таблицами лиц и организаций — система на нём поднимется и будет
> работоспособной.
>
> Дампы лежат в `dist/` и **не попадают в git**: каталог в `.gitignore`, чтобы
> один неосторожный `git add -A` не сделал их публичными навсегда.

Учётные записи входят в дамп вместе с паролями отправителя. После
восстановления смените их: `python -m scripts.seed_access --reset-passwords`.
Документация API: `http://127.0.0.1:8100/docs`.

У каждого загрузчика есть `--dry-run`: он выполняет все вставки и проверки
по-настоящему и в конце откатывает транзакцию.

## Демонстрационные учётные записи

Создаются командой `python -m scripts.seed_access`.

| Логин | Роль | Территория | Зачем нужна |
|---|---|---|---|
| `admin` | Администратор | все | Полный доступ, правка моделей риска |
| `analyst` | Аналитик | Алматинская область | Обычный рабочий сценарий |
| `analyst.karasay` | Аналитик | Карасайский район | Проверка территориального ограничения: Талгарский район недоступен |
| `manager` | Руководитель | все | Просмотр и отчёты |
| `viewer` | Просмотр | Алматинская область | Минимальные права |

**Паролей в коде нет.** Они генерируются и печатаются один раз, при создании
записи; в базе хранится только хеш Argon2id. Одинаковый пароль во всех
развёртываниях переживает и демонстрацию, и приёмку, и оказывается в проде.
Перевыдать: `python -m scripts.seed_access --reset-passwords`.

## Сверка с книгами-источниками

Расчёт риска выполняется независимо от книг: в книге 8.5 все 10 240 формул лежат
без кэша и читаются как `None`, а расчётные листы 8.4, 8.6 и 8.7 — статические
экспорты вовсе без формул. Значения книг сохраняются отдельно и используются
только как контрольные.

Контрольные значения сошлись до последней цифры:

| Слой | Распределение уровней (низкий / средний / высокий / критический) | Прочее |
|---|---|---|
| **8.5** Субсидии | **3344 / 67 / 2 / 0** | Σ субсидий **67 535 553 445 ₸**, max R = 72.095 |
| **8.4** Госзакупки | **89 / 172 / 43 / 48** + **3 серых** | max балл 67.083; все 48 критических — по категории A |
| **8.3** Бюджет | **187 / 52 / 0 / 1** | max **75.0** — ровно пол балла, а не расчёт |
| **8.6** ГЧП | **1066 / 141 / 44 / 8** + **64 серых** | max 85.727 |
| **8.6** Экспертиза | **4221 / 567 / 54** | критических и серых нет |
| **8.7** Организации | **3645 серых + 23 критических** | max балл 93.3 при полноте 40.9 % |

Три серых договора и 64 серых проекта — **не сбой загрузки**, а зафиксированные
аудитом дефекты и пробелы источников, воспроизведённые как есть.

Организация с максимальным баллом 93.3 официально имеет уровень «нет данных», а
критическими становятся организации с баллом 42.2 и ниже. Это не ошибка:
балл при полноте 41 % не основание для вывода, а признак лжепредприятия —
основание. Подробности — в [`docs/risk-models.md`](docs/risk-models.md).

## Что система сознательно не делает

Полный перечень — в [`docs/assumptions-and-gaps.md`](docs/assumptions-and-gaps.md).
Главное:

* **Не показывает границы из комплекта.** Набор GADM, пришедший в исходниках,
  отражает деление до реформы 2022 года и распространяется по лицензии,
  запрещающей перераспространение, вопреки заявленному в архиве public domain.
  Используются границы OpenStreetMap под ODbL 1.0 с обязательной атрибуцией.
* **Не достраивает КАТО.** Единственного официального ключа стыковки территорий
  нет ни в одной книге и почти нет в геоданных. Связывание идёт по названиям
  через справочник алиасов; неопознанное название не угадывается, а попадает в
  отчёт о качестве.
* **Не выводит организации на карту.** В книге 8.7 нет ни района, ни адреса, ни
  координат, ни КАТО. Придумать 3668 объектам координаты означало бы поместить
  их в точки, которых источник не знает.
* **Не выдумывает учредителей.** Состава учредителей нет ни в одной книге. Тип
  связи объявлен, функция вывода написана, отчёт печатает ноль, и отдельный тест
  это закрепляет — чтобы никто потом не заполнил его догадками.
* **Не показывает ноль вместо «нет данных».** Карточка «Аналитические материалы
  и меры» предусмотрена ТЗ, но такой сущности в приложенных данных не существует
  вовсе. Она честно говорит «нет данных» и объясняет причину.
* **Не восстанавливает прежние значения полей при откате импорта.** Откат
  отзывает актуальность версии и ничего не удаляет, но построчной истории в
  схеме нет — это записано в коде как ограничение.
* **Не закрывает слой 8.7 полностью.** Модель обеспечена данными на **41 %**:
  работают 5 индикаторов из 13, у остальных нет публичного API. Неподключённые
  индикаторы видны в карточке как «не измерено» — именно они объясняют серый
  уровень.
